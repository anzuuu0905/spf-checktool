# Sprint 5: モックv2 本実装反映

## 概要

- **目的**: `docs/design-mock/mockup-2026-05-18-v2.html` の確定仕様を本実装に反映し、スコア数値ベースの旧UIから「信号機3色＋達成率」ベースの新UIへ全面刷新する。
- **前提**: Sprint 1〜4 完了済み（DNS解析・診断ロジック・Google Forms統合まで実装済み）、Phase 0（リサーチ・議論・社長承認）完了済み。
- **本質的な変更点**: (1) DKIM検出網の3倍化（10→28セレクタ並列）、(2) スコア数値の完全廃止と「3項目中N項目クリア」表記への一本化、(3) 煽り語彙・技術用語の全削除、(4) CTA を「無料でメール相談する」単一に集約。
- **デプロイ**: `main` 直プッシュで GitHub Actions による自動デプロイ（既存ワークフロー利用）。

Sprint 4 で導入した Google Forms 統合は維持しつつ、UI 構造・文言・CTA を全面的にモックv2 と一致させる。煽り表現を排除して「迷惑メールフォルダ」観点の落ち着いたトーンに統一し、技術用語を生活者語彙に置換することで、SPF/DKIM/DMARC を知らないユーザーでも自然に読める診断ツールへリポジショニングする。

---

## 実装スコープ

### 1. DKIMセレクタリスト拡充（10 → 28個）

`src/parsers/dkim-parser.js` の `checkCommonSelectors()` を以下28個に拡充する：

```
default, google, selector1, selector2, k1, k2, k3, s1, s2, s1024, s2048,
mail, mail2, dkim, mte1, mte2, hs1, hs2, zmail,
fm1, fm2, fm3, protonmail, protonmail2, protonmail3, zendesk1, zendesk2, kl
```

カバー対象: Google Workspace, Microsoft 365, エックスサーバー, MailChimp, SendGrid, HubSpot, Zoho, Fastmail, ProtonMail, Zendesk, Klaviyo 等の主要メールサービス。

### 2. DNS並列実行

`src/api/dns-resolver.js` の `getDKIMRecords()` を `Promise.all` で並列化。28セレクタ同時クエリで全体待ち時間を 1 秒以内に抑える。

### 3. UI構造刷新（モックv2 移植）

- スコア数値（`0/10`, `7/10`, `10/10` など）を**完全廃止**。
- 信号機3色 + ラベル「安全 / 要改善 / 要対応」 + 達成率「3項目中N項目クリア」に置換。
- 4パターン判定:

| パターン | 条件 | 色 | ラベル | 達成率 |
|---|---|---|---|---|
| 1 | SPF ✅ + DKIM ✅ + DMARC (p≥quarantine) ✅ | 緑 | 安全 | 3項目中3項目クリア |
| 2 | SPF ✅ + DKIM ✅ かつ（DMARC なし or p=none） | 黄 | 要改善 | 3項目中2項目クリア |
| 3 | SPF ✅ のみ | 赤 | 要対応 | 3項目中1項目クリア |
| 4 | すべて不合格 | 赤 | 要対応 | 3項目中0項目クリア |

### 4. テキスト全文をモックv2と一致

- ヒーロー、結果カード explain、why-block、verdict-summary、教育アコーディオン、すべてモックv2 の文言と100%一致。
- DKIM 未検出時の説明文（必須）:
  > 「Google Workspace・Microsoft 365・エックスサーバー など主要なメールサービス28種類の設定で確認しましたが、見つかりませんでした。」
  > 注記: 「※ さくらインターネット・ロリポップ・Amazon SES 等の一部サービスは独自形式の設定を使うため、ドメインだけでは判定できないことがあります。」
- DKIM 状態テキスト・aria-label は「**未検出**」（SPF/DMARC の「未設定」と区別する）。
- 禁止語彙（煽り）: 「危険」「攻撃」「最終防衛ライン」「被害発生のリスク」「ほぼ未設定」。
- 禁止語彙（技術用語）: 「統合判定」「運用ルール」「p=none」「正規送信元」。
- 「迷惑メールフォルダ」観点で文言統一。

### 5. CTA変更

- 各パターンの結果直後と末尾の **2 箇所** に「無料でメール相談する」ボタン。
- リンク: `https://forms.gle/fBEiPFxku2cdnwtL8`
- 属性: `target="_blank" rel="noopener"`
- サブノート（「営業電話は一切いたしません」等）は**不要**。
- 旧 CTA は**削除**: 「定期チェックを設定」「より詳細な診断」「Outlook 義務化バッジ」。

### 6. 教育コンテンツ

- モックv2 のアコーディオン4セクション構造を維持: (1) 仕組み、(2) なぜ必要、(3) 優先順位、(4) FAQ。
- パターン2（黄）/3（赤）/4（赤）では「なぜ必要」のみ初期展開、他は閉じる。パターン1（緑）は全閉。
- 「p=none」は「監視モード」に置換済み（モック反映）。

## NG基準（即不合格）

以下が1件でも該当した場合、Evaluator は無条件で不合格を出す：

- スコア数値（例: `0/10`, `7/10`, `10/10`）が画面の1箇所でも残存
- 煽り語彙（「危険」「攻撃」「最終防衛ライン」「被害発生のリスク」「ほぼ未設定」）のいずれかが残存
- 技術用語（「統合判定」「運用ルール」「p=none」「正規送信元」）のいずれかが残存
- DKIM の状態テキストが「未設定」のまま（「未検出」に置換されていない）
- 「無料でメール相談する」ボタンが結果直後または末尾のいずれかに存在しない
- Google Form の URL が `https://forms.gle/fBEiPFxku2cdnwtL8` 以外
- DKIM セレクタが 28 個に達していない
- DNS 呼び出しが直列実行のまま（`Promise.all` を使用していない）
- main プッシュ前に Evaluator のテストが Pass していない

### シナリオ4: 全不合格（赤・0/3）
