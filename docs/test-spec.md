## テストデータ

### 正常系ドメイン

| ドメイン | SPF | DKIM | DMARC | 説明 |
|---|---|---|---|---|
| `google.com` | ✅ | ✅ | ✅ | 全設定済み |
| `github.com` | ✅ | ✅ | ❌ | DMARC未設定 |
| `example.com` | ✅ | ❌ | ❌ | SPFのみ |
| `test.invalid` | ❌ | ❌ | ❌ | 全未設定 |

### 異常系ドメイン

| 入力値 | 期待エラー |
|---|---|
| 空欄 | 「ドメインを入力してください」 |
| 全角スペースのみ | 「ドメインを入力してください」 |
| 全角文字（日本語除く） | 「正しいドメインを入力してください」 |
| `thisisnotarealdomainxyz123.com` | 「ドメインが見つかりません」 |

### エッジケース

| ケース | 説明 | 期待動作 |
|---|---|---|
| ネットワークエラー | DNS API 接続失敗 | 「接続エラーが発生しました。しばらく待ってから再試行してください」 |
| レート制限 | DNS API のレート制限 | 「診断が混み合っています。少し待ってから再試行してください」 |
| 長いドメイン名 | 253文字の有効なドメイン | 正常に診断 |
| IDN（国際化ドメイン） | `日本語.jp` | Punycode 変換後に診断 |

## E2Eテスト項目（Sprint 1 受け入れ基準 1:1 マッピング）

### 画面表示（5項目）
- [ ] ページタイトル「メール配信診断ツール」
- [ ] キャッチコピー「あなたの会社のメールは正しく届いていますか？」
- [ ] プレースホルダー「例: example.com」
- [ ] 診断ボタンの青色スタイル
- [ ] レスポンシブ: 375px / 768px / 1440px

### ドメイン入力と診断開始（6項目）
- [ ] ドメイン入力 → 診断ボタンクリック
- [ ] ローディング表示が3秒以内に開始
- [ ] ローディング中は診断ボタン無効化
- [ ] 診断完了後に診断ボタン再有効化
- [ ] 空欄でエラー「ドメインを入力してください」
- [ ] 不正形式でエラーメッセージ

### SPF診断結果（4項目）
- [ ] 設定済み: 緑チェックマーク + 「設定済み」
- [ ] 未設定: 赤警告 + 「未設定」
- [ ] 未設定時の説明文表示
- [ ] 「SPFとは何ですか？」クリックで説明展開

### DKIM診断結果（4項目）
- [ ] 設定済み: 緑チェックマーク + 「設定済み」
- [ ] 未設定: オレンジ注意 + 「未設定」
- [ ] 未設定時の説明文表示
- [ ] 「DKIMとは何ですか？」クリックで説明展開

### DMARC診断結果（4項目）
- [ ] 設定済み: 緑チェックマーク + 「設定済み」
- [ ] 未設定: 黄色情報 + 「未設定」
- [ ] 未設定時の説明文表示
- [ ] 「DMARCとは何ですか？」クリックで説明展開

### CTAと誘導（5項目）
- [ ] 未設定項目ある場合: 「専門家に相談する」ボタン表示
- [ ] 相談ボタン上のメッセージ「設定には専門的な知識が必要です」
- [ ] 相談ボタンクリック → 外部フォーム遷移（URL検証）
- [ ] 全設定済み: 「問題ありません」緑メッセージ
- [ ] 全設定済みでも「さらに詳しく診断したい方はこちら」リンク

### エラーハンドリング（3項目）
- [ ] 存在しないドメイン → 「ドメインが見つかりません」
- [ ] ネットワークエラー → 「接続エラーが発生しました。しばらく待ってから再試行してください」
- [ ] レート制限 → 「診断が混み合っています。少し待ってから再試行してください」

### パフォーマンス要件（5項目）
- [ ] 診断処理3秒以内
- [ ] Lighthouse Performance ≥ 90
- [ ] Lighthouse Accessibility ≥ 90
- [ ] Lighthouse Best Practices ≥ 90
- [ ] First Contentful Paint (FCP) ≤ 1.8秒
